## 2 报送生成流程

```
DWS层汇总完成 (T日 07:00)
    │
    ▼
ADS监管表生成 (07:00-07:30)
    ├── ads_reg_1104_*  ←  dws_fin_* + dws_loan_* + dws_dep_*
    ├── ads_reg_east_*  ←  dwd_sat_* + dwd_hub_*
    └── ads_reg_aml_*   ←  dwd_sat_risk_alert
    │
    ▼
数据校验 (07:30-07:45)
    ├── 完整性校验 (必填字段非空)
    ├── 码值校验 (EAST/1104码值规范)
    ├── 跨表一致性校验
    └── 金额勾稽校验
    │
    ├── PASS → 导出文件 (CSV/TXT/XML)
    │          └── ads_reg_submit_log (记录报送)
    └── FAIL → 修复后重跑
```

## 3 关键报送要求

### 3.1 EAST校验规则

| 校验码 | 校验内容 | 级别 |
|--------|---------|------|
| E001 | 必填字段非空 | HIGH |
| E002 | 日期格式YYYYMMDD | HIGH |
| E003 | 金额>=0 | MEDIUM |
| E004 | 码值在字典范围内 | HIGH |
| E005 | 客户号在个人基础信息表存在 | CRITICAL |
| E006 | 账号与客户号的关联正确 | CRITICAL |

### 3.2 数据导出示例

```sql
-- EAST T_50_1 数据导出
SELECT 
    account_no,
    DATE_FORMAT(trans_date, 'yyyyMMdd') AS trans_date,
    DATE_FORMAT(trans_time, 'HHmmss') AS trans_time,
    trans_seq,
    CASE dr_cr_flag WHEN 'D' THEN '1' WHEN 'C' THEN '2' END AS dr_cr_flag,
    CAST(trans_amt * 100 AS BIGINT) AS trans_amt,  -- 转为分
    ...
FROM ads_reg_east_t_50_1
WHERE report_period = '202606';
```
