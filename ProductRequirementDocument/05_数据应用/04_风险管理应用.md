## 1 五大风险类型管理

| 风险类型 | 核心应用 | 数据源 | 时效 |
|---------|---------|--------|------|
| **信用风险** | 客户评级、五级分类、拨备计提 | `dws_loan_classification_summary`, `ads_risk_customer_rating` | T+1 |
| **操作风险** | 柜面操作监控、权限审计 | `dwd_sat_transaction` (渠道=COUNTER) | T+1 |
| **市场风险** | 利率重定价缺口、久期分析 | `dws_dep_term_structure`, `dws_fin_daily_pnl` | T+1 |
| **流动性风险** | 头寸管理、LCR/NSFR | `dws_risk_liquidity` | 实时+ T+1 |
| **合规风险** | 反洗钱(AML)、名单筛查 | `dwd_sat_risk_alert` | 实时+ T+1 |

## 2 信用风险管理

### 2.1 PD模型输入数据

```sql
-- 计算客户违约概率(PD)的输入特征
SELECT 
    customer_hash_key,
    overdue_days_max_12m,      -- 近12月最大逾期天数
    debt_to_income_ratio,      -- 负债收入比
    credit_history_years,      -- 信用历史年限
    loan_cnt_active,           -- 当前活跃贷款笔数
    credit_card_utilization,   -- 信用卡额度使用率
    recent_credit_inquiry_cnt, -- 近6月征信查询次数
    industry_risk_score,       -- 行业风险评分(对公)
    ...
FROM dws_cust_daily_summary;
```

### 2.2 五级分类与拨备

```
贷款五级分类 → 拨备计提比例
  正常类      → 1%
  关注类      → 2%
  次级类      → 25%
  可疑类      → 50%
  损失类      → 100%

拨备覆盖率 = 贷款损失准备 / 不良贷款余额 ≥ 150% (监管要求)
```

## 3 操作风险管理

**监控对象**：柜面操作中的异常行为

| 监控项 | 检测逻辑 | 数据来源 |
|--------|---------|---------|
| 频繁冲正 | 同一柜员1h内冲正>3笔 | `dwd_sat_transaction` (trans_type=REVERSAL) |
| 越权操作 | 交易金额超柜员权限 | 对比柜员权限表 |
| 非营业时间操作 | 非营业时间的大额交易 | `dwd_sat_transaction` (渠道=COUNTER) |
| 密码试错 | 同一账户短时间内多次密码错误 | 网银登录事件 |

## 4 流动性风险管理

| 指标 | 计算 | 监管要求 |
|------|------|---------|
| **流动性比例** | 流动资产/流动负债 | ≥ 25% |
| **LCR** | HQLA/未来30天净现金流出 | ≥ 100% |
| **NSFR** | 可用稳定资金/所需稳定资金 | ≥ 100% |
| **核心负债依存度** | 核心负债/总负债 | ≥ 60% |
| **存贷比** | 贷款余额/存款余额 | 监测指标 |
