## 1 标签体系架构

```
客户标签体系 (5大类, 50+标签)
│
├── ① 基础属性 (静态/准静态)
│   ├── 年龄、性别、职业、收入、学历
│   ├── 地区、开户时间、客户年限
│   └── 企业规模、行业、注册资本(对公)
│
├── ② 资产标签 (日更新)
│   ├── AUM分层: UHNW/HNW/AFFLUENT/MASS/LONG_TAIL
│   ├── 存款偏好: 活期型/定期型/通知型
│   └── 理财偏好: 保守/稳健/进取
│
├── ③ 行为标签 (日更新)
│   ├── 活跃度: 高频/中频/低频/沉默
│   ├── 渠道偏好: 柜面/网银/手机/POS
│   ├── 交易时段: 日间/夜间/周末
│   └── 跨境偏好: 有/无
│
├── ④ 信用标签 (日/月更新)
│   ├── 信用评分: 300-850
│   ├── 逾期历史: 有/无, M1/M2/M3+
│   └── 负债水平: 高/中/低
│
└── ⑤ 价值标签 (日更新)
    ├── 客户价值评分: 1-100
    ├── 生命周期: 新增/成长/成熟/衰退/流失
    ├── 交叉销售潜力: HIGH/MEDIUM/LOW
    └── 流失概率: 0-1
```

## 2 标签计算逻辑

### 2.1 AUM分层

```sql
SELECT customer_hash_key,
    CASE 
        WHEN total_aum >= 10000000 THEN 'UHNW'
        WHEN total_aum >= 5000000  THEN 'HNW'
        WHEN total_aum >= 500000   THEN 'AFFLUENT'
        WHEN total_aum >= 100000   THEN 'MASS'
        ELSE 'LONG_TAIL'
    END AS aum_level
FROM ads_cust_360_view WHERE dt = '${yesterday}';
```

### 2.2 客户价值评分

```sql
-- 加权评分模型
SELECT customer_hash_key,
    ROUND(
        COALESCE(aum_score, 0) * 0.30 +
        COALESCE(active_score, 0) * 0.25 +
        COALESCE(product_score, 0) * 0.20 +
        COALESCE(credit_score, 0) * 0.15 +
        COALESCE(loyalty_score, 0) * 0.10
    , 0) AS value_score
FROM ...;
```

### 2.3 生命周期判断

```sql
CASE 
    WHEN open_years < 1 THEN 'NEW'
    WHEN open_years BETWEEN 1 AND 3 AND active_level = 'HIGH' THEN 'ACTIVE'
    WHEN open_years > 3 AND active_level IN ('HIGH','MEDIUM') THEN 'MATURE'
    WHEN active_level = 'LOW' AND churn_probability < 0.5 THEN 'DECLINE'
    WHEN active_level = 'SILENT' OR churn_probability >= 0.5 THEN 'CHURN'
END AS life_cycle
```

## 3 标签输出与应用

| 应用场景 | 使用标签 | 输出方式 |
|---------|---------|---------|
| **精准营销** | 价值评分+交叉销售潜力+渠道偏好 | `ads_cust_360_view` → CRM系统 |
| **风险预警** | 信用评分+逾期历史+流失概率 | `ads_cust_360_view` → 风控系统 |
| **客户关怀** | 生命周期+活跃度+资产变化 | 推送提醒给客户经理 |
| **产品推荐** | next_best_product + 产品持有情况 | 手机银行推荐位 |
