## 1 实时大屏指标

### 1.1 指标清单

| 指标 | 计算方式 | 刷新 | 存储 |
|------|---------|------|------|
| 今日交易量 | COUNT(txn) 当天累计 | 5s | Redis String |
| 实时交易金额 | SUM(amount) 当天累计 | 5s | Redis String |
| 在线客户数 | COUNT(DISTINCT customer) 5min窗口 | 5s | Redis Set |
| 交易成功率 | SUCCESS/总数 5min窗口 | 5s | Redis String |
| 渠道分布 | 按channel分组笔数/金额 | 5s | Redis Hash |
| 风险告警滚动 | 最新50条告警 | 实时 | Redis List |

### 1.2 Redis Key设计

```
screen:txn:cnt          → "128456"            (String, TTL=10s)
screen:txn:amt          → "3286542100.00"     (String, TTL=10s)
screen:online:users     → {user1, user2, ...} (Set, TTL=300s)
screen:channel:pie      → {MOBILE:45, IB:20, ATM:15, POS:12, COUNTER:8} (Hash)
screen:alert:list       → [alert1, alert2, ...] (List, LRANGE 0 49)
```

## 2 告警通知设计

### 2.1 告警级别

| 级别 | 色标 | 通知方式 | 响应要求 |
|------|------|---------|---------|
| **CRITICAL** | 红色 | 钉钉+短信+电话 | 立即处理(5分钟内) |
| **HIGH** | 橙色 | 钉钉+邮件 | 15分钟内处理 |
| **MEDIUM** | 黄色 | 邮件 | 1小时内处理 |
| **LOW** | 蓝色 | 仅记录 | 无需立即处理 |

### 2.2 告警通知模板

```json
{
  "alert_id": "ALT20260615-001234",
  "alert_type": "LARGE_AMOUNT",
  "risk_level": "HIGH",
  "alert_time": "2026-06-15T14:30:15",
  "customer": "李**",
  "account": "6222****0123",
  "amount": "¥650,000.00",
  "channel": "手机银行",
  "rule": "R001-大额交易监控",
  "action_required": "请确认该交易是否为客户本人操作"
}
```

## 3 告警处理流程

```
Flink CEP 检测 ──→ 生成告警 ──→ 写入 Kafka (bank.dq.alert)
                                    │
                        ┌───────────┴───────────┐
                        ▼                       ▼
                  存入MySQL              钉钉/邮件通知
                  (dwd_sat_risk_alert)         
                        │
                        ▼
                  人工审核处理
                  ├── 确认风险 → 升级案例(阻断/上报)
                  ├── 误报 → 标记FALSE_POSITIVE
                  └── 正常 → 标记RESOLVED
```
