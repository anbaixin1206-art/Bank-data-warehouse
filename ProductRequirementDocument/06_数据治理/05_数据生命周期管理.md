## 1 各层保留策略

| 层 | 在线存储 | 降冷策略 | 归档策略 | 最终删除 |
|-----|---------|---------|---------|---------|
| **STG** | 3-7天 | — | — | 每日自动删除过期分区 |
| **ODS** | 30天 | — | — | 30天后自动删除 |
| **DWD** | 1年(在线HDFS) | 1年后迁至冷存储(HDFS归档目录) | 3年后归档至Hadoop Archive(HAR) | 5年后可选删除 |
| **DWS** | 2年(在线HDFS) | 2年后迁至冷存储 | 5年后归档至HAR | 7年后可选删除 |
| **ADS(监管)** | 永久保留 | — | 5年后归档至HAR | 永久保留 |
| **ADS(非监管)** | 1年 | — | 1年后归档 | 3年后删除 |
| **临时表/ETL中间表** | 24小时 | — | — | 作业完成后自动清理 |

## 2 分区清理脚本

```sql
-- 每日 00:30 执行：清理过期分区

-- STG清理(7天前)
ALTER TABLE stg_core_t_transaction DROP IF EXISTS PARTITION (dt='${date_7_days_ago}');

-- ODS清理(30天前)
ALTER TABLE ods_core_t_transaction DROP IF EXISTS PARTITION (dt='${date_30_days_ago}');

-- DWD归档(1年前)
-- Step1: 移动至冷存储
INSERT OVERWRITE TABLE dwd_sat_transaction_archive PARTITION (dt='${date_1_year_ago}')
SELECT * FROM dwd_sat_transaction WHERE dt = '${date_1_year_ago}';
-- Step2: 删除在线分区
ALTER TABLE dwd_sat_transaction DROP IF EXISTS PARTITION (dt='${date_1_year_ago}');
```

## 3 冷热数据分层存储

```
热数据 (在线HDFS)
  ├── STG/ODS: 所有数据 (短期)
  ├── DWD: dt > DATE_SUB(today, 365)
  ├── DWS: dt > DATE_SUB(today, 730)
  └── ADS: dt > DATE_SUB(today, 365)

冷数据 (HDFS冷目录 /data/cold/)
  ├── DWD: 1-3年前数据
  └── DWS: 2-5年前数据

归档数据 (Hadoop Archive .har)
  ├── DWD: 3-5年前
  └── DWS: 5-7年前
```
