## 1 质量规则体系

| 规则类型 | 检查内容 | 适用层 | 严重级别 | 示例 |
|---------|---------|--------|---------|------|
| **完整性** | NULL值比例 | ODS/DWD | HIGH | 客户姓名空值率 < 0.1% |
| **唯一性** | 主键重复 | DWD | CRITICAL | HUB表HASH_KEY唯一 |
| **准确性** | 数据类型/格式 | ODS | MEDIUM | 手机号格式`^1[3-9]\d{9}$` |
| **一致性** | 跨表关联一致 | DWD/DWS | HIGH | 存款余额=SUM(账户明细) |
| **及时性** | 数据延迟 | ODS | MEDIUM | ingest_time-业务时间<2h |
| **有效性** | 码值范围 | DWD | MEDIUM | 交易类型在字典内 |
| **参照完整性** | 外键有效 | DWD | CRITICAL | LINK表HASH_KEY在HUB中存在 |

## 2 质量检查SQL模板

```sql
-- 完整性检查
SELECT '${table}' AS tbl, 'completeness' AS rule,
    SUM(CASE WHEN ${col} IS NULL THEN 1 ELSE 0 END) AS null_rows,
    ROUND(SUM(CASE WHEN ${col} IS NULL THEN 1 END)*100.0/COUNT(*), 4) AS null_rate,
    CASE WHEN null_rate > ${threshold} THEN 'FAIL' ELSE 'PASS' END AS result
FROM ${table} WHERE dt = '${yesterday}';

-- 唯一性检查
SELECT '${table}' AS tbl, 'uniqueness' AS rule,
    CASE WHEN COUNT(*) <> COUNT(DISTINCT ${pk}) THEN 'FAIL' ELSE 'PASS' END AS result
FROM ${table} WHERE dt = '${yesterday}';

-- 参照完整性检查
SELECT '${link_table}' AS tbl, 'ref_integrity' AS rule,
    SUM(CASE WHEN h.${hub_key} IS NULL THEN 1 ELSE 0 END) AS orphan_rows
FROM ${link_table} l
LEFT JOIN ${hub_table} h ON l.${fk} = h.${hub_key}
WHERE l.dt = '${yesterday}' AND h.dt = '${yesterday}';
```

## 3 质量门禁

```
ETL作业完成
    │
    ▼
质量检查(DQ Check)
    ├── CRITICAL FAIL → 阻断下游 + 立即告警(钉钉+邮件)
    ├── HIGH FAIL     → 阻断下游 + 告警
    ├── MEDIUM FAIL   → 告警 + 继续(标记WARN)
    └── ALL PASS      → 继续下游
```
