## 1 分期总览

```
一期 (核心基础)        二期 (全业务覆盖)        三期 (增强与优化)
2-3个月               3-4个月                  2-3个月
─────────────────────  ──────────────────────  ──────────────────────
基础设施部署           全业务域模型             实时增强
+ 核心域数仓模型       + 数据治理体系           + 性能优化
+ 离线ETL链路          + 调度体系               + 监控完善
+ 首个应用(驾驶舱)     + 全部应用               + 文档完善
```

---

## 2 一期：核心基础（2-3个月）

**目标**：搭建基础设施 + 核心业务域 + 验证端到端链路

### 基础设施

| 任务 | 说明 | 预计工作量 |
|------|------|-----------|
| Kafka 部署 | 单节点Kafka，配置Topic | 1天 |
| Canal 部署 | 部署+配置MySQL Binlog订阅 | 1天 |
| DolphinScheduler 部署 | 单节点DS | 1天 |
| Redis 部署 | Docker化部署 | 0.5天 |
| Flume 部署(可选) | 文件采集 | 0.5天 |

### 数仓建模

| 域 | 范围 | 预计工作量 |
|-----|------|-----------|
| 客户域 | HUB/SAT/LINK全部 | 3天 |
| 机构域 | 全部 | 1天 |
| 产品域 | 全部 | 1天 |
| 渠道域 | 全部 | 1天 |
| 存款域 | 全部 | 2天 |
| 支付域 | 交易统一模型 | 3天 |
| DWS汇总 | 客户/存款/支付3个域 | 2天 |
| ADS | 管理驾驶舱 | 1天 |

### ETL开发

| 任务 | 预计工作量 |
|------|-----------|
| DataX采集配置(5个核心源系统) | 3天 |
| STG→ODS ETL | 2天 |
| DWD ETL (5个域) | 5天 |
| DWS ETL (3个域) | 2天 |
| ADS ETL (驾驶舱) | 1天 |

### 一期交付物

```
✅ 基础设施就绪(Kafka/Canal/DS/Redis)
✅ 5个核心域模型(DWD)
✅ 3个域汇总(DWS)
✅ 管理驾驶舱(ADS)
✅ 离线日批链路可运行
✅ 数据从源系统到驾驶舱端到端验证通过
```

## 3 二期：全业务覆盖（3-4个月）

**目标**：覆盖全部13个主题域 + 数据治理体系 + 全部应用

| 模块 | 内容 | 预计工作量 |
|------|------|-----------|
| DWD扩展 | 信用卡/理财/国际结算/风控/监管/财务 6个域 | 10天 |
| DWS扩展 | 信用卡/理财/风控/财务 4个域 | 5天 |
| ADS扩展 | 监管报送/客户画像/风险管理/BI | 8天 |
| 数据治理 | 元数据/质量/血缘/安全/生命周期 | 8天 |
| 调度体系 | DolphinScheduler完整DAG编排 | 5天 |
| 非日批 | 小时批/周批/月批/季批 | 3天 |

---

## 4 三期：增强与优化（2-3个月）

| 模块 | 内容 | 预计工作量 |
|------|------|-----------|
| 实时增强 | Flink CEP风控完整规则集 | 5天 |
| 实时对账 | Flink SQL支付-核心双流对账 | 3天 |
| 实时大屏 | Redis+WebSocket实时大屏 | 3天 |
| 性能优化 | 分区/分桶/统计信息/查询优化 | 5天 |
| 监控完善 | 全链路监控+自动化告警 | 3天 |
| 文档完善 | 操作手册 + 配置指南 | 3天 |
| 压力测试 | 模拟数据生成 + 性能验证 | 3天 |

---

## 5 总工作量预估

| 阶段 | 自然月 | 核心工作 |
|------|--------|---------|
| 一期 | 1-3月 | 基础设施 + 5核心域 + 离线链路 + 驾驶舱 |
| 二期 | 4-7月 | 13域全覆盖 + 治理 + 调度 + 全部应用 |
| 三期 | 8-10月 | 实时增强 + 优化 + 监控 + 文档 |

**总计**：约 8-10 个月（1人全职投入）
