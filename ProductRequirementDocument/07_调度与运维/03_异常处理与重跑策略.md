## 1 异常分级

| 级别 | 定义 | 处理方式 | 示例 |
|------|------|---------|------|
| **P0 致命** | 核心表数据丢失/大面积失败 | 立即告警(电话)+阻断全部下游+人工介入 | HDFS故障、HUB_CUSTOMER全量失败 |
| **P1 严重** | 单个域/重要表失败 | 自动重试3次→失败告警(钉钉+邮件)+阻断该域下游 | DWD某域ETL失败 |
| **P2 一般** | 非关键表失败 | 自动重试2次→失败告警(邮件)+下游继续 | BI宽表失败 |
| **P3 提示** | DQ检查WARN | 仅记录+告警(邮件) | NULL率略高于阈值 |

## 2 重试策略

| 作业类型 | 重试次数 | 重试间隔 | 退避策略 |
|---------|---------|---------|---------|
| DataX采集 | 3 | 5min | 固定间隔 |
| Hive ETL | 2 | 2min | 固定间隔 |
| Spark ETL | 2 | 3min | 固定间隔 |
| DQ检查 | 1 | 1min | 固定间隔 |
| Shell脚本 | 0 | — | 不重试(直接告警) |

## 3 重跑策略

### 3.1 单表重跑

```bash
# 重跑指定表指定日期
hive -e "INSERT OVERWRITE TABLE dwd_hub_customer PARTITION (dt='2026-06-14') SELECT ...;"
```

**影响范围**：仅下游依赖该表的作业

### 3.2 域级重跑

```bash
# 重跑存款域DWD→DWS→ADS全链路
bash /mnt/d/bigdata-lab/bank-data-warehouse/scripts/rerun_domain.sh deposit 2026-06-14
```

### 3.3 全链路回溯

```bash
# 指定日期范围全链路重跑
bash /mnt/d/bigdata-lab/bank-data-warehouse/scripts/rerun_full.sh 2026-06-01 2026-06-14
```

## 4 数据延迟处理

当源系统数据延迟到达时：

```
1. 标记LATE_ARRIVE分区
   ALTER TABLE ods_core_t_transaction ADD PARTITION (dt='2026-06-14', late='true');

2. 次日补跑昨日分区
   INSERT OVERWRITE TABLE dwd_sat_transaction PARTITION (dt='2026-06-14')
   SELECT ... FROM ods_core_t_transaction WHERE dt='2026-06-14';

3. 下游依赖自动感知 (DolphinScheduler补数功能)
```

## 5 故障恢复检查清单

```
□ HDFS 是否正常 (hdfs dfsadmin -report)
□ Hive Metastore 是否正常 (beeline -e "SHOW DATABASES;")
□ MySQL 是否正常 (mysql -e "SELECT 1;")
□ Kafka 是否有积压 (consumer group lag)
□ DataX canal 是否正常运行
□ 磁盘空间是否充足 (df -h /data)
```
