## 1 监控体系

| 监控维度 | 监控对象 | 采集方式 | 告警方式 |
|---------|---------|---------|---------|
| **基础设施** | CPU/内存/磁盘/网络 | WSL2系统命令 | 钉钉 |
| **服务状态** | HDFS/Hive/Spark/Flink/MySQL/Kafka | 端口检查+进程检查 | 钉钉+邮件 |
| **作业运行** | ETL成功率/耗时/数据量 | DolphinScheduler API | 钉钉 |
| **数据质量** | DQ检查PASS/FAIL率 | DQ脚本输出 | 钉钉+邮件 |
| **数据时效** | 采集延迟/ETL延迟 | 时间戳差值 | 钉钉 |

## 2 服务健康检查脚本

```bash
#!/bin/bash
# check_services.sh — 服务健康检查

check_port() {
    ss -tlnp | grep -q "$1" && echo "✅ $2" || echo "❌ $2"
}

check_process() {
    jps | grep -q "$1" && echo "✅ $1" || echo "❌ $1"
}

check_port 3306 "MySQL"
check_port 9000 "HDFS NameNode"
check_port 9870 "HDFS WebUI"
check_port 9083 "Hive Metastore"
check_port 10000 "HiveServer2"
check_port 9092 "Kafka"
check_port 6379 "Redis"

check_process "NameNode"
check_process "DataNode"
check_process "RunJar"  # Metastore/HiveServer2
```

## 3 告警规则

| 告警 | 触发条件 | 级别 | 通知方式 |
|------|---------|------|---------|
| 服务宕机 | 端口/进程不存在 | P0 | 钉钉+电话 |
| 磁盘使用率>80% | `df -h` | P1 | 钉钉+邮件 |
| ETL失败 | 作业状态=FAILED | P1 | 钉钉+邮件 |
| ETL超时 | 耗时>历史均值*2 | P2 | 钉钉 |
| DQ FAIL | CRITICAL/HIGH规则失败 | P1 | 钉钉+邮件 |
| 采集延迟>30min | ingest_time-业务时间>30min | P2 | 邮件 |
| Kafka积压>10000 | Consumer Lag > 10000 | P1 | 钉钉 |
