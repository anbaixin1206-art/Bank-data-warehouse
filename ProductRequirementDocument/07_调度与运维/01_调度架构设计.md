## 1 DolphinScheduler 架构

```
DolphinScheduler (待部署)
│
├── Master Server    — 任务调度 + DAG解析 + 状态管理
├── Worker Server    — 任务执行 (多个Worker节点)
├── API Server       — REST API
└── Zookeeper        — 集群协调 (单节点环境下可内嵌)
```

**与本项目关系**：
- Master/Worker 部署在 WSL2 内
- 通过 Web UI 管理作业和查看DAG
- 所有 DataX/Spark/Hive/Shell 作业交由 DS 统一编排

## 2 调度周期设计

| 周期类型 | 触发器 | 典型作业 |
|---------|--------|---------|
| **日批** | 每日 00:00 | STG→ODS→DWD→DWS→DQ→ADS 全链路 |
| **小时批** | 每小时 | 风控指标更新、头寸监控 |
| **15分钟批** | 每15分钟 | DataX核心交易增量采集 |
| **周批** | 每周一 08:00 | 周度汇总、周报 |
| **月批** | 每月1日 06:00 | 月度汇总、监管月报、结息 |
| **季批** | 每季初 06:00 | 1104季报、EAST季报 |

## 4 调度配置示例

```yaml
# DolphinScheduler DAG 流程定义 (示意)
workflow:
  name: bank_daily_batch
  schedule: "0 0 * * *"
  global_params:
    yesterday: "${system.biz.date}"
  
  tasks:
    - name: stg_core_account
      type: DATAX
      depends: []
      
    - name: ods_core_account
      type: HIVE
      depends: [stg_core_account]
      
    - name: dwd_hub_customer
      type: HIVE
      depends: [ods_core_account, ods_ecif_party]
      
    - name: dwd_sat_customer_info
      type: HIVE
      depends: [dwd_hub_customer]
      
    - name: dws_cust_daily_summary
      type: SPARK
      depends: [dwd_sat_customer_info, dwd_sat_account_bal]
      
    - name: dq_check_completeness
      type: SHELL
      depends: [dws_cust_daily_summary]
      
    - name: ads_mgmt_kpi
      type: HIVE
      depends: [dq_check_completeness]
      
    - name: notify_success
      type: SHELL
      depends: [ads_mgmt_kpi]
```
