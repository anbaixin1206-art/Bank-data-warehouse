### 源系统分类

| 类别 | 系统 | 数据库类型 | 数据特征 |
|------|------|----------|---------|
| **交易核心** | 核心银行、支付网关 | Oracle / MySQL | 高频交易、强一致性、ACID |
| **业务管理** | 信贷管理、信用卡、理财 | Oracle / MySQL | 业务流程状态、中频变更 |
| **渠道系统** | 网银手机银行、ATM/POS | MySQL / 文件 | 用户行为、日志型、高频 |
| **管理支撑** | 客服、反洗钱 | MySQL | 管理流程、中低频 |
| **基础数据** | 财务总账、客户ECIF | Oracle | 主数据、低频变更、高一致性 |

## 2 源系统数据字典

### 2.1 ① 核心银行系统 (CORE-BANK / Oracle)

**系统说明**：全行最核心的交易处理系统，管理所有账户和资金交易。

| 表名 | 表中文名 | 采集方式 | 日增量 | 关键字段 |
|------|---------|---------|--------|---------|
| `T_CUSTOMER` | 客户信息表 | 全量+增量 | 万级 | `CUSTOMER_ID`, `CUST_NAME`, `ID_TYPE`, `ID_NO`, `MOBILE`, `CUST_TYPE`, `CREATE_DATE` |
| `T_ACCOUNT` | 账户主表 | 全量+增量 | 十万级 | `ACCOUNT_NO`, `CUSTOMER_ID`, `ACCT_TYPE`, `CURRENCY`, `OPEN_DATE`, `STATUS`, `BRANCH_ID` |
| `T_ACCOUNT_BALANCE` | 账户余额表 | 全量快照 | 百万级 | `ACCOUNT_NO`, `BALANCE`, `AVAIL_BALANCE`, `FROZEN_AMT`, `LAST_UPDATE` |
| `T_TRANSACTION` | 交易流水表 | 增量 | 千万级 | `TRANS_ID`, `ACCOUNT_NO`, `TRANS_TYPE`, `TRANS_AMT`, `DR_CR_FLAG`, `TRANS_TIME`, `CHANNEL`, `TELLER_ID`, `OPP_ACCOUNT` |
| `T_INTEREST` | 计息明细表 | 增量 | 百万级 | `INTEREST_ID`, `ACCOUNT_NO`, `BASE_AMT`, `RATE`, `INTEREST_AMT`, `INT_DATE` |

## 3 源系统数据特征分析

### 3.1 数据量级分级

| 级别 | 日增量 | 典型系统 | 采集策略 |
|------|--------|---------|---------|
| **极高** (>1千万/天) | 核心交易流水、支付流水、电子渠道埋点 | Binlog 实时 + 增量 |
| **高** (百万-千万/天) | 信用卡消费、凭证明细、还款记录 | 增量 + 每日快照 |
| **中** (万-百万/天) | 账户信息、放款记录、理财交易 | 增量 + 每日快照 |
| **低** (<1万/天) | 客户信息、产品信息、反洗钱案例 | 每日全量快照 |

### 3.2 数据变更特征

| 变更模式 | 典型表 | DWD 层处理方式 |
|---------|--------|--------------|
| **追加型** | 交易流水、支付流水 | Satellite (事实) 直接追加 |
| **快照型** | 余额表、科目余额 | Satellite (属性) 每日快照 |
| **变更型** | 客户信息、账户状态 | Satellite (属性) 拉链处理 (SCD Type 2) |
| **主数据类型** | 产品、机构、科目 | Hub + Satellite (属性) |
