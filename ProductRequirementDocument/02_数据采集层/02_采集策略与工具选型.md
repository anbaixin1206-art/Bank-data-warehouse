## 2 工具选型详解

### 2.1 DataX — 离线批量采集

**适用场景**：Oracle/MySQL 数据库的批量离线采集

**核心能力**：
- 支持 20+ 种数据源（RDBMS、NoSQL、文件、HDFS）
- 全量 + 增量模式（自定义 SQL）
- 并发 Channel 控制
- 失败重试 + 脏数据阈值

**在本项目中的角色**：
- 所有 Oracle 源系统的全量+增量采集
- MySQL 源系统的全量快照采集（非实时表）
- 小时级增量调度

**DataX 作业模板 — Oracle 增量采集**：

```json
{
  "job": {
    "setting": {
      "speed": {
        "channel": 5,
        "bytes": 10485760
      },
      "errorLimit": {
        "record": 1000,
        "percentage": 0.01
      }
    },
    "content": [
      {
        "reader": {
          "name": "oraclereader",
          "parameter": {
            "username": "${ods_reader_user}",
            "password": "${ods_reader_pass}",
            "connection": [
              {
                "jdbcUrl": ["jdbc:oracle:thin:@//${CORE_HOST}:1521/COREDB"],
                "querySql": [
                  "SELECT * FROM T_TRANSACTION 
                   WHERE TRANS_TIME >= TO_DATE('${start_time}', 'YYYY-MM-DD HH24:MI:SS')
                   AND TRANS_TIME < TO_DATE('${end_time}', 'YYYY-MM-DD HH24:MI:SS')"
                ]
              }
            ]
          }
        },
        "writer": {
          "name": "hdfswriter",
          "parameter": {
            "defaultFS": "hdfs://localhost:9000",
            "fileType": "orc",
            "path": "/data/stg/core/t_transaction/dt=${date}/hh=${hour}",
            "fileName": "t_transaction",
            "column": [
              {"name": "TRANS_ID", "type": "STRING"},
              {"name": "ACCOUNT_NO", "type": "STRING"},
              {"name": "TRANS_TYPE", "type": "STRING"},
              {"name": "TRANS_AMT", "type": "DECIMAL"},
              {"name": "DR_CR_FLAG", "type": "STRING"},
              {"name": "TRANS_TIME", "type": "TIMESTAMP"},
              {"name": "CHANNEL", "type": "STRING"},
              {"name": "TELLER_ID", "type": "STRING"},
              {"name": "OPP_ACCOUNT", "type": "STRING"}
            ],
            "writeMode": "append",
            "compress": "SNAPPY"
          }
        }
      }
    ]
  }
}
```

### 2.2 Canal — MySQL Binlog 实时采集

**适用场景**：MySQL 源系统的实时变更数据捕获 (CDC)

**核心能力**：
- 伪装 MySQL Slave，订阅 Binlog
- 支持 INSERT / UPDATE / DELETE 事件
- 原生 Kafka 输出（Canal Kafka Connector）
- 支持数据过滤和转换

**在本项目中的角色**：
- 支付网关流水实时采集
- 信用卡消费交易实时采集
- 网银/手机银行登录/交易事件实时采集
- 理财系统交易实时增量

**Canal 配置示例**：

```properties
# canal.properties
canal.id = 1
canal.ip = 127.0.0.1
canal.port = 11111
canal.zkServers = 127.0.0.1:2181
canal.serverMode = kafka
kafka.bootstrap.servers = 127.0.0.1:9092

# instance.properties (以支付网关为例)
canal.instance.mysql.slaveId = 1001
canal.instance.master.address = 127.0.0.1:3306
canal.instance.dbUsername = canal_user
canal.instance.dbPassword = canal_pass
canal.instance.filter.regex = pay_db\\.t_payment_flow,pay_db\\.t_settlement
canal.instance.filter.black.regex = pay_db\\.t_pay_channel_log

# Kafka 输出配置
canal.mq.topic = bank.ods.pay
canal.mq.dynamicTopic = bank.ods.pay.${schema}.${table}
canal.mq.partition = 0
canal.mq.partitionsNum = 8
canal.mq.partitionHash = pay_db.t_payment_flow:trans_id
```

**Canal 输出消息格式**：

```json
{
  "data": [
    {
      "pay_id": "PAY20260615100000001",
      "order_no": "ORD20260615987654321",
      "payer_acct": "6222021234567890123",
      "payee_acct": "6228489876543210987",
      "amount": "50000.00",
      "pay_channel": "IBPS",
      "pay_status": "SUCCESS",
      "create_time": "2026-06-15 10:00:05",
      "complete_time": "2026-06-15 10:00:12"
    }
  ],
  "database": "pay_db",
```

### 2.3 Flume — 文件/日志采集

**适用场景**：非数据库数据源的实时文件采集

**核心能力**：
- TAILDIR Source：监控目录，增量采集新增文件/追加内容
- Memory/File Channel：缓存保证不丢数据
- Kafka Sink：直接输出到 Kafka

**在本项目中的角色**：
- ATM 交易文件采集
- POS 交易文件采集
- 网银用户行为日志采集

**Flume 配置示例 — ATM 交易文件**：

```properties
# flume-atm.conf
agent.sources = atm_source
agent.channels = atm_channel
agent.sinks = atm_sink

# Source: 监控 ATM 文件目录
agent.sources.atm_source.type = TAILDIR
agent.sources.atm_source.filegroups = atm_fg
agent.sources.atm_source.filegroups.atm_fg = /data/atm/transactions/atm_transactions_*.csv
agent.sources.atm_source.positionFile = /data/flume/position/atm_position.json
agent.sources.atm_source.batchSize = 1000

# Channel: 文件通道保证持久性
agent.channels.atm_channel.type = file
agent.channels.atm_channel.checkpointDir = /data/flume/checkpoint/atm
agent.channels.atm_channel.dataDirs = /data/flume/data/atm
agent.channels.atm_channel.capacity = 1000000
agent.channels.atm_channel.transactionCapacity = 5000

# Sink: 输出到 Kafka
agent.sinks.atm_sink.type = org.apache.flume.sink.kafka.KafkaSink
agent.sinks.atm_sink.kafka.topic = bank.ods.atmpos.atm_transaction
agent.sinks.atm_sink.kafka.bootstrap.servers = 127.0.0.1:9092
agent.sinks.atm_sink.kafka.flumeBatchSize = 1000
agent.sinks.atm_sink.kafka.producer.acks = 1
agent.sinks.atm_sink.kafka.producer.compression.type = snappy
```

## 3 采集链路总览

```
【离线批采集链路（T+1）】
  Oracle/MySQL 源库
    │
    ├── DataX (JSON配置 → DolphinScheduler调度)
    │     ├─ 全量快照（每日 00:00）
    │     ├─ 增量SQL（每小时/15min）
    │     └─ 输出: HDFS STG 层 (/data/stg/<system>/<table>/dt=<date>/)
    │
    └── Hive LOAD DATA
          └─ STG → ODS (INSERT OVERWRITE/INTO)

【实时采集链路】
  MySQL 源库
    │
    └── Canal Server (Binlog 订阅)
          │
          ├── Kafka Topic
          │     ├─ Flink Consumer → 实时 DWD (风控/对账/大屏)
          │     └─ Spark Streaming → ODS 准实时更新
          │
          └── 直接写入 (不走 Kafka 的场景)
                └─ HBase/Redis (实时指标缓存)

【文件采集链路】
  ATM/POS 文件服务器 → Flume TAILDIR
    └── Kafka Topic
          └── Flink/Spark → HDFS → ODS
```

## 4 采集作业管理

### 4.1 作业命名规范

```
格式: {tool}_{source_system}_{table}_{mode}

示例:
  datax_core_t_customer_full        # DataX-核心银行-客户表-全量
  datax_core_t_customer_inc         # DataX-核心银行-客户表-增量
  datax_loan_t_contract_full        # DataX-信贷-合同表-全量
  canal_pay_t_payment_flow          # Canal-支付网关-支付流水
  flume_atm_transactions            # Flume-ATM交易文件
```

### 4.2 作业监控指标

| 指标 | 说明 | 告警阈值 |
|------|------|---------|
| `采集延迟` | 业务时间 - 采集完成时间 | 增量 > 10min，全量 > 2h |
| `数据量偏差` | 今日采集行数 vs 昨日采集行数 | 偏差 > 30% |
| `数据量绝对值` | 采集行数 | 核心交易 < 1千万/天 |
| `采集成功率` | 成功作业数 / 总作业数 | < 95% |
| `脏数据率` | 格式错误/空值超限记录数占比 | > 0.1% |
