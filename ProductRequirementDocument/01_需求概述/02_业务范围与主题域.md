## 2 第一层：参与方（Party）

### 2.1 ① 客户域

**业务定位**：全行统一的客户信息中心，构建客户 360° 视图。

| 子域 | 核心实体 | 覆盖内容 |
|------|---------|---------|
| 个人客户 | 客户编号、姓名、证件、联系方式、地址 | KYC 信息、客户等级、职业、年收入、教育程度 |
| 对公客户 | 企业编号、企业名称、法人代表、注册资本 | 工商信息、股权结构、经营范围、企业规模 |
| 客户关系 | 关系类型、关联方、关系强度 | 担保关系、关联人、集团隶属、家庭成员 |
| 客户画像 | 标签类型、标签值、标签来源 | 客户分层(AUM)、行为标签、风险标签、价值评分 |
| ECIF 统一视图 | 跨系统客户识别、客户归并 | 核心银行/信用卡/理财系统的客户映射 |

**与其他域的关系**：客户域为所有业务域提供客户维度。

### 2.2 ② 机构域

**业务定位**：全行机构层级与组织架构管理。

| 子域 | 核心实体 | 覆盖内容 |
|------|---------|---------|
| 机构管理 | 机构编号、机构名称、机构类型、上级机构 | 总行→一级分行→二级分行→支行→网点 |
| 部门管理 | 部门编号、部门名称、所属机构 | 业务部门、管理部门、支持部门 |
| 员工管理 | 员工编号、姓名、岗位、所属机构/部门 | 柜员、客户经理、信贷审批员、风控专员 |

**与其他域的关系**：机构域为所有业务域提供归属机构维度。

## 3 第二层：产品与渠道

### 3.1 ③ 产品域

**业务定位**：全行金融产品目录与产品属性管理。

| 产品大类 | 产品子类 | 关键属性 |
|---------|---------|---------|
| **存款产品** | 活期存款、定期存款、通知存款、大额存单、结构性存款 | 起存金额、利率、期限、提前支取规则 |
| **贷款产品** | 个人住房贷款、消费贷款、经营贷款、对公流动资金贷款、银团贷款 | 贷款额度、利率类型(LPR/固定)、期限、还款方式 |
| **支付产品** | 行内转账、跨行转账、快捷支付、代收代付 | 限额、手续费、到账时效 |
| **信用卡产品** | 普卡、金卡、白金卡、钻石卡、联名卡、分期卡 | 额度范围、年费、权益、积分规则 |
| **理财产品** | 货币基金、债券理财、净值型理财、信托计划、保险代销 | 风险等级、预期收益、起购金额、封闭期 |
| **国际结算** | 信用证、托收、汇款、保函 | 币种、期限、适用规则(UCP600等) |

### 3.2 ④ 渠道域

**业务定位**：客户触达银行的各类渠道及其交易特征。

| 渠道 | 渠道类型 | 关键特征 |
|------|---------|---------|
| 柜面 | 物理渠道 | 高柜(现金)、低柜(非现金)、综合柜员 |
| 手机银行 | 电子渠道 | 7×24、生物识别、消息推送 |
| 个人网银 | 电子渠道 | 大额交易、U盾认证 |
| 企业网银 | 电子渠道 | 多级授权、批量交易 |
| ATM | 自助设备 | 取款、存款、转账、查询 |
| POS | 收单终端 | 消费、预授权、撤销 |
| 第三方合作 | 外部渠道 | 微信支付、支付宝、银联 |

## 4 第三层：核心业务过程

### 4.1 ⑤ 存款域

| 子域 | 核心业务过程 | 关键事实 |
|------|------------|---------|
| 账户管理 | 开户、销户、挂失冻结、解冻、账户变更 | 账户状态、开户日期、销户日期 |
| 资金交易 | 现金存入、现金支取、转账存入、转账支取 | 交易金额、交易时间、交易渠道、柜员号 |
| 利息计算 | 结息、利息计提、利息税 | 计息积数、利率、利息金额 |

### 4.2 ⑥ 贷款域

| 子域 | 核心业务过程 | 关键事实 |
|------|------------|---------|
| 授信管理 | 授信申请、信用评估、额度审批、额度变更 | 授信金额、额度类型、有效期 |
| 合同管理 | 合同签订、合同变更、合同终止 | 贷款金额、利率、期限、还款方式 |
| 放款 | 贷款发放、受托支付 | 放款金额、放款日期、收款账户 |
| 还款 | 正常还款、提前还款、逾期还款 | 还款金额、本金、利息、罚息 |
| 贷后管理 | 五级分类、贷后检查、催收 | 分类结果、分类日期、逾期天数 |

### 4.3 ⑦ 支付域

| 子域 | 核心业务过程 | 关键事实 |
|------|------------|---------|
| 行内转账 | 本行账户间资金划转 | 转出账户、转入账户、金额、用途 |
| 跨行支付 | 大额支付(HVPS)、小额支付(BEPS)、超级网银(IBPS) | 支付渠道、对手行、清算状态 |
| 清算结算 | 银联清算、网联清算、头寸管理 | 清算金额、清算状态、对账结果 |
| 代收代付 | 代发工资、代扣水电费、批量支付 | 批次号、总笔数、总金额 |

### 4.4 ⑧ 信用卡域

| 子域 | 核心业务过程 | 关键事实 |
|------|------------|---------|
| 卡片管理 | 发卡、激活、挂失、补卡、销卡 | 卡号、有效期、CVV、激活日期 |
| 消费交易 | 刷卡消费、线上消费、分期消费 | 消费金额、商户、MCC码、是否分期 |
| 账单管理 | 账单生成、对账、争议处理 | 账单日期、还款日、最低还款额 |
| 还款 | 全额还款、最低还款、分期还款 | 还款金额、还款渠道、是否逾期 |

### 4.5 ⑨ 理财域

| 子域 | 核心业务过程 | 关键事实 |
|------|------------|---------|
| 产品管理 | 产品创建、募集、成立、到期 | 产品规模、募集期、开放日 |
| 交易 | 认购、申购、赎回、分红 | 交易金额、份额、净值 |
| 持仓管理 | 持仓查询、收益计算 | 持有份额、当前净值、浮动盈亏 |
| 适当性管理 | 风险评估、产品匹配、双录 | 风险等级、投资期限偏好 |

### 4.6 ⑩ 国际结算域

| 子域 | 核心业务过程 | 关键事实 |
|------|------------|---------|
| 信用证 | 开证、通知、修改、交单、议付 | 信用证金额、效期、适用规则 |
| 汇款 | SWIFT汇出汇款、汇入汇款 | 汇款币种、金额、起息日、中间行 |
| 托收 | 进口代收、出口托收 | 托收金额、付款期限、交单条件 |
| 保函 | 开立保函、修改、索赔 | 保函类型、金额、效期 |

## 5 第四层：管理与合规

### 5.1 ⑪ 风控域

| 子域 | 覆盖内容 | 时效要求 |
|------|---------|---------|
| 反欺诈 | 实时交易欺诈检测、账户异常行为监控 | 实时(<500ms) |
| 信用风险 | 客户信用评分、违约概率(PD)、违约损失率(LGD)、风险缓释 | T+1 |
| 操作风险 | 柜面操作监控、权限异常、冲正抹账分析 | T+1 |
| 合规检查 | 大额交易识别、可疑交易模式匹配、制裁名单筛查 | 实时+T+1 |
| 流动性风险 | 头寸监控、LCR/NSFR 指标、流动性缺口 | 实时(头寸) + T+1 |

### 5.2 ⑫ 监管域

| 报送类型 | 报送对象 | 频次 | 关键内容 |
|---------|---------|------|---------|
| 1104 非现场监管 | 国家金融监管总局 | 月/季/半年/年 | 资产负债、损益、存贷款、流动性、资本充足（40+ 张报表） |
| EAST 监管标准化 | 国家金融监管总局 | 月/季 | 全量明细数据（存款/贷款/信用卡/理财/支付，60+ 张表） |
| 大集中(反洗钱) | 中国人民银行反洗钱中心 | T+1(大额)/10日(可疑) | 大额交易报告、可疑交易报告 |
| 二代征信 | 中国人民银行征信中心 | 月/日报 | 个人/企业借贷信息、担保信息 |
| 利率报备 | 中国人民银行 | 月 | LPR执行情况、存贷款利率 |

### 5.3 ⑬ 财务域

| 子域 | 覆盖内容 |
|------|---------|
| 总账管理 | 科目余额、记账凭证、账务核对 |
| 损益管理 | 营业收入、营业支出、利润、ROA/ROE |
| 管理会计 | 责任中心、成本分摊、内部资金转移定价(FTP)、产品盈利分析 |

## 6 域间依赖关系

```
                    ┌──────────────┐
                    │ ① 客户域     │
                    │ ② 机构域     │
                    └──────┬───────┘
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
    ┌──────────┐   ┌──────────┐   ┌──────────┐
    │ ③ 产品域 │   │ ④ 渠道域 │   │   ...    │
    └────┬─────┘   └────┬─────┘   └──────────┘
         └──────────────┼──────────────────────┐
                        ▼                      │
    ┌──────────────────────────────────────────┼──┐
    │  ⑤存款  ⑥贷款  ⑦支付  ⑧信用卡  ⑨理财  ⑩国际结算 │
    └──────────────────────────┬───────────────┘──┘
                               ▼
    ┌──────────────────────────────────────────────┐
    │         ⑪ 风控域  ──→  ⑫ 监管域               │
    │                      ⑬ 财务域                │
    └──────────────────────────────────────────────┘
```

> **建模顺序原则**：先建参与方和产品/渠道等公共维度，再建各业务过程明细，最后汇总到风控/监管/财务域。
